# cnsite-ng-pages

本仓库用于归档和发布多个独立的 HTML 页面。这些页面彼此之间没有直接关联，各自独立运行，不针对任何特定域名或具体网站。

## 项目简介

`cnsite-ng-pages` 是一个以存档为目的的静态页面集合。仓库中的每个 HTML 文件均可单独访问，适合用于存放各类示例页面、临时展示内容或历史资料归档。所有页面均为纯前端实现，不依赖后端服务。

## 目录说明

```
├── index.html          # 仓库根目录下的首页（可选）
├── pages/              # 存放主要 HTML 页面的目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：所有独立的 HTML 页面存放于此，每个文件对应一个独立页面。
- `assets/`：页面共享或单独使用的资源文件，按类型组织。

## 页面归档说明

- 每个页面均为独立的 HTML 文件，可直接在浏览器中打开运行。
- 页面之间无耦合，不共享状态或数据。
- 归档内容可能包含示例、测试页面、历史版本或临时展示用途。
- 部分页面可能引用外部资源（如 CDN 上的库），请确保网络连通性。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面。
- 删除或修改已有页面时，建议在提交信息中注明原因。
- 欢迎通过 Issues 提交建议或报告问题，但请注意本仓库不针对任何具体网站或服务。
- 如需引用仓库中的页面，请直接使用原始文件链接，不建议二次分发。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `pages/` 目录下的任意 HTML 文件。
3. 也可通过 GitHub Pages 或其他静态托管服务部署后访问。
